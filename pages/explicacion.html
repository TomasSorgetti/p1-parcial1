Explicación parcial programacón I

Se realizó una aplicación que está diseñada para gestionar un catálogo de discos musicales, permitiendo a los usuarios agregar y visualizar discos, así como sus pistas y stock disponible

Estructura del programa

La aplicación sigue una arquitectura modular con la intención de aplicar una notoria separación de responsabilidades.

Flujo principal

El flujo comienza con la interacción del usuario realizando un evento, que es capturada por el controlador, que se encarga de recibir la solicitud inicial, para luego derivarla al servicio, el cual se encarga de realizar las tareas de la aplicación e interactuar con las entidades.

Controlador

El controlador se encarga de interactuar con el usuario para pedirle datos, para luego derivarlos al servicio y notificar al usuario en caso de que haya habido algún problema.

Servicios

El servicio se encarga de realizar las tareas de la aplicación e interactuar con las entidades de los discos, pistas y catálogo. Es un puente entre los datos del disco y las operaciones que necesita realizar el controlador.

Entidades

Las entidades son clases que representan los objetos principales de la aplicación, definen la estructura y el comportamiento de los discos, las pistas que contienen, y el catálogo donde se almacenan.

Loop de repeticiones

Cuando se pide cada dato para un disco, se deriva a una clase validadora y se asegura de que el dato sea correcto. Si el dato es incorrecto, se repite el proceso mediante una función recursiva, y no un bucle, por que reduce la cantidad de código y facilita la lectura(aunque se puede implementar un bucle, que tiene mejor rendimiento).

Uso de try catch

En la aplicación se utiliza un try catch para manejar los errores que puedan ocurrir al interactuar con el usuario, mostrando una alerta cuando ocurre un error, o para repetir una función si el usuario ingresa un dato incorrecto.
